## 오늘 공부한 것
TS 책을 읽다가 "타입을 붙이는 방법이 여러 개던데 실무에선 뭘 많이 쓰지?"가 궁금해져서, 타입을 지정하는 방식들을 정리했다.

## 내가 가졌던 질문
- 배열 타입 쓸 때 제너릭(`Array<T>`)이랑 축약형(`T[]`) 중 뭘 많이 쓰나?
- `const num: number = 1234` 처럼 타입을 매번 붙이나?
- `var ad = 1234` 하면 TS가 알아서 number인 걸 아나?
- 타입 지정 방법 3가지가 정확히 뭐뭔가?
- 그중 제일 많이 쓰는 건 추론인가?
- 추론/명시/단언은 각각 언제 쓰나?

## 정리 (Q&A 핵심)

### 배열 타입: `Array<number>` vs `number[]`
둘은 완전히 같은 타입이다. 문법만 다름.

```ts
const nums: Array<number> = [1, 2, 3]; // 제너릭 표기
const nums: number[] = [1, 2, 3];       // 축약형
```

실무에선 **`number[]` 축약형이 사실상 표준**. 제너릭 표기는 유니온처럼 축약형이 애매할 때 골라 쓴다.

```ts
const a: (string | number)[] = []; // 괄호 필요, 살짝 헷갈림
const a: Array<string | number> = []; // 이게 더 명확
```

### 변수엔 타입을 매번 붙이지 않는다 (추론)
TS는 타입 추론이 있어서 값을 보고 타입을 알아낸다. 그래서 자명한 건 안 붙인다.

```ts
const num: number = 1234; // 틀린 건 아닌데 군더더기
const num = 1234;          // 이렇게 씀 (number 로 추론됨)
```

### 선언 키워드와 상관없이 추론은 된다
`var / let / const` 어느 걸 써도 오른쪽 값을 보고 추론한다. 다만 `var`는 옛날 문법이라 요즘은 `const`(기본), 바뀌면 `let`을 쓴다.

`const`는 값이 안 바뀌니 더 좁게(리터럴 타입) 추론된다.

```ts
let a = 1234;    // a: number
const b = 1234;  // b: 1234  ← 값 자체로 고정 (리터럴 타입)
```

### 타입을 지정하는 3가지 방법

| 방법 | 문법 | 누가 정함 |
|------|------|-----------|
| 명시(Annotation) | `: number` | 나 |
| 추론(Inference) | (안 적음) | TS |
| 단언(Assertion) | `as number` | 나 (강제) |

### 언제 뭘 쓰나 — "값이 옆에 있으면 추론, 없으면 명시"
- **변수**는 `= 1234`처럼 값이 옆에 있으니 → 추론
- **함수 매개변수**는 정의 시점엔 값이 없으니 → 명시 (안 붙이면 `any`)

```ts
const num = 1234;               // 변수 → 추론
function add(a: number, b: number) { // 매개변수 → 명시
  return a + b;
}
```

(처음에 이걸 반대로 이해했다가 바로잡았다. 변수=추론, 매개변수=명시.)

### 단언 `as` — TS가 못 알아챌 때 바로잡는 수단
"이건 네가 생각하는 것 말고 이 타입이야"라고 못박는 것.

```ts
// DOM: getElementById 는 HTMLElement | null 로 넓게 잡힘
const input = document.getElementById("email") as HTMLInputElement;
input.value; // as 로 input 이라 알려줘서 접근 가능

// API 응답: JSON.parse 는 any → 모양을 지정
const data = JSON.parse(response) as User;
```

주의: `as`는 실제로 바꾸는 게 아니라 **TS 눈만 속인다.** 거짓말하면 런타임에 터진다. 그래서 꼭 필요할 때만 아껴 쓴다.

### 빈도 감각
| 방법 | 얼마나 | 주로 어디서 |
|------|:---:|------|
| 추론 | 제일 많음 | 초기값 있는 변수 |
| 명시 | 그다음 | 함수 매개변수·반환값, 초기값 없는 `let` |
| 단언(`as`) | 가끔 | TS가 못 알아챌 때만 |

## 아직 헷갈리는 것 / 다음에 볼 것
- 커스텀 타입을 만드는 쪽(`type` / `interface` / `enum`)은 이번에 이름만 짚고 넘어감. 다음에 `type` vs `interface` 차이를 따로 파기.
- 함수 반환 타입을 명시할지 추론에 맡길지는 팀 스타일 문제라던데, 실제 기준을 더 알아보기.

## 한 줄 요약
변수는 추론, 함수 매개변수는 명시, `as`는 최소한 — 이 감각이 TS 타입의 기본.
